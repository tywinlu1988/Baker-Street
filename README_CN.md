# 🔍 贝克街 221B — 福尔摩斯多视角分析框架

> *"When you have eliminated the impossible, whatever remains, however improbable, must be the truth."*
> *—— Sherlock Holmes, The Sign of the Four*

**贝克街（Baker Street）** 是一个基于 Claude Code 的多视角分析引擎。它从福尔摩斯原著中的 7 个角色提炼出 7 种认知原型，以独立 Agent 并行分析同一问题，再通过冲突挖掘、盲区检测和行动路径合成，产出任何单一模型都无法独自达到的洞察深度。

这不是一个"列出优缺点"的提示词模板，而是一个**思维引擎** —— 它的价值不在于输出格式有多漂亮，而在于能否让用户看到自己（以及模型）原本看不到的东西。

## 1. 核心洞察：为什么需要这个框架

### 问题的本质

Fable 5、Opus 4.8 —— 这些模型已经能给出非常结构化的回答。但任何一个有经验的思考者都能察觉到：**结构化不等于有洞察**。

模型的"分析"本质上是知识重排（knowledge rearrangement）。它把训练数据中相关的内容提取、整理、格式化输出。这个过程产出的东西看起来很好，但缺少两样关键的东西：

1. **真正的发现（Discovery）** —— 模型不会主动推翻你的初始假设。它倾向于在你的问题框架内给出最优排列，而不是指出"你问的问题本身就是错的"。
2. **认知冲突（Cognitive Friction）** —— 任何深刻的思考都包含自我怀疑、自我反驳、在不同价值观之间权衡。单一模型的一次性输出天然缺少这种张力。

### 这个框架如何破解

贝克街框架的核心机制是 **强制多视角并行独立分析**：

```
不是：一个模型 → 一次回答 → "嗯，看起来不错"
而是：7 个独立认知原型 → 并行分析 → 挖掘冲突 → 发现盲区 → 合成洞察
```

关键设计在于"独立"二字。每个角色 Agent 在分析时**不知道其他角色的结论**。这意味着：

- 它们会基于不同的前提、不同的价值观优先级、不同的方法论，得出可能**相互矛盾**的结论
- 这些矛盾不是 bug，而是框架最核心的价值来源 —— 它们暴露了问题中被单一视角掩盖的复杂性
- 汇总层不是做"取平均值"的调和，而是**放大冲突**，让用户在矛盾中看到更完整的图景

### 三大结构优势（v0.3.x）

A/B 测试证明了多 Agent 管道在纯推理质量上并不优于精心设计的单体 prompt。但管道在三个维度上具有单体 prompt 无法复制的结构优势：

1. **反迎合（Anti-Sycophancy）** —— LLM 被训练成倾向于同意用户。研究 Agent **必须**寻找反证：至少 2 条事实必须质疑用户隐含的假设。典型事实库中包含 20-35% 的反证事实。在报告中以"反迎合得分"呈现。
2. **上下文扩展（Context Expansion）** —— 单次 prompt 受限于一个上下文窗口。多 Agent 研究可以突破：2-3 个研究 Agent 各自产出 15-30 条事实 → 合并为 30-60+ 条可验证声明的事实库。通过 `--research-depth` 控制广度。
3. **视角发散（Perspective Divergence）** —— 单体 prompt 在同一个上下文中模拟 7 个视角——认知平滑不可避免。独立 Agent 产出真正发散的视角，可通过声明唯一率（CUR）量化度量。

### 定量证据

我们设计了 Framework Gain（框架增益）指标来衡量这一价值：

```
Framework Gain = 框架输出的平均新颖性 / 裸模型回答的新颖性
```

v0.1 的目标值是 ≥ 1.5 —— 即框架至少比裸模型多产出 50% 的新颖洞察。这不是凭空猜测，而是通过 5 个标准化测试用例 + LLM-as-Judge 评分来验证的。

## 2. 设计哲学

### 为什么是福尔摩斯？

福尔摩斯原著之所以适合作为认知原型来源，不是因为它们是"推理小说"，而是因为柯南·道尔创造了一系列**认知风格高度差异化的角色**：

- 福尔摩斯的演绎法 vs 华生的经验归纳
- 迈克罗夫特的系统思维 vs 雷斯垂德的务实证据
- 莫里亚蒂的博弈视角 vs 艾琳·艾德勒的情感智慧

每个角色不是文学形象，而是一种**认知原型（Cognitive Archetype）** —— 一种稳定的、可迁移的思维方法论。框架提取的是原型，不是人设。

### 为什么必须并行独立？

如果让一个模型依次"扮演"不同角色，后面的角色会知道前面的角色说了什么。这会破坏视角的独立性 —— 角色之间的认知冲突会被无意识地平抑。

并行 + 独立 = 真实的分歧。这才是多视角分析的价值前提。

### 冲突是 Feature，不是 Bug

一个快速的健康检查：**如果所有角色一致同意，要么问题太简单，要么角色没有真正用力。**

框架在汇总阶段做的第一件事就是冲突挖掘（Conflict Mining）。不是找"你说 Rust 我说 Go"这种表面分歧，而是找三种深层矛盾：

| 冲突类型 | 含义 | 示例 |
|----------|------|------|
| **假设冲突** | 两个角色对同一变量做了相反的假设 | Holmes 假设"团队能学会 Rust"，Lestrade 假设"学习曲线是决定性瓶颈" |
| **权重冲突** | 角色对同一组因素赋予了不同的优先级 | Moriarty 把安全风险排第一，Watson 把开发效率排第一 |
| **解读冲突** | 同一事实，不同的因果解读 | "增长停滞" → Adler 解读为定位问题，Lestrade 解读为执行问题 |

### 诚实比全面更重要

每个角色都有**盲区声明（Blind Spot Declaration）** —— 明确承认自己天然忽视什么。不是假装全能，而是诚实地标注自己的认知边界。

汇总层会检查：在所有角色的盲区之外，还有哪些维度**没有任何角色覆盖**？这些"沉默的维度"往往就是最有价值的发现。

### 成本自觉

框架不是 7 个角色全量轰炸。v0.1 的成本控制策略：

- **按需选角**：根据问题类型自动匹配 2-4 个最相关的角色
- **深度分档**：`--depth quick`（2 角色）、`standard`（3-4）、`deep`（7）
- **冲突回注帽**：最多只对 2 对冲突进行角色间的互相回应

## 3. 七个认知原型

### 🔍 夏洛克·福尔摩斯 —— 演绎推理

> *"When you have eliminated the impossible, whatever remains, however improbable, must be the truth."*

**认知立场**：在仔细观察所有细节之前不形成任何理论。生成多个假设，然后系统地排除不可能的，剩下的无论多不可能，就是真相。

**核心追问**：
1. 其他人忽略了什么细节？
2. 哪些假设已经被证据排除了？
3. 证据链是否闭合，是否存在缺口？
4. 如果从结果倒推，什么前提必须成立？
5. **没有什么**这件事本身说明了什么？（"那只没叫的狗"）

**思维方法**：从事实出发，不允许自己先有结论再找证据。特别注意异常、不一致、和"不太对劲"的地方。当两个解释都能成立时，更简单的那个通常是正确的。

**天生盲区**：情感因素、社会常规、道德直觉。会系统性地低估"人"的维度。

**最适合**：技术决策、事实调查、需要排除假想敌的场景。

---

### 📋 约翰·华生 —— 常识归纳

> *"What seems extraordinary is often the result of ordinary causes."*

**认知立场**：最平凡的答案往往是最正确的。用日常经验和普通人的判断力来检验那些"聪明过头"的理论。

**核心追问**：
1. 对这件事最日常、最无趣的解释是什么？
2. 过去的经验（我的和别人的）告诉我们什么？
3. 一个理性的普通人会怎么做？
4. 我们是不是被一个漂亮的理论带偏了，而忽略了眼前简单的事实？
5. 一个纯粹逻辑的方法会错过的、眼见为实的人性现实是什么？

**思维方法**：用真实世界的经验和实践智慧检验每一个推论。相信对人的直觉 —— 在看人方面通常是对的。注意那些偏离日常现实太远的理论。终极检验："如果我喝酒时跟朋友这么说，他会信吗？"

**天生盲区**：罕见事件、极端情况、反直觉的真相。对"平凡解释"的偏好可能导致错过真正不平凡的事情。

**最适合**：需要现实检验的场景、产品决策、用户行为分析。

---

### 🧠 迈克罗夫特·福尔摩斯 —— 系统思维

> *"The problem is never isolated. Every fact is a node in a larger system."*

**认知立场**：所有问题都嵌套在更大的系统中。永远寻找结构性的、政策层面的根因而非最近的诱因。思考但不行动。

**核心追问**：
1. 这个问题嵌套在什么更大的系统中？
2. 哪些间接的、远端的、二阶的因素在真正驱动局面？
3. 结构或制度层面的根因是什么 —— 不只是最近触发了什么？
4. 如果系统中什么都不改变，这个问题会复发吗？
5. 谁从现行安排中获益？这种激励结构如何维持现状？

**思维方法**：在分析任何组件之前先绘制系统全貌。识别反馈循环、激励结构、和结构性约束。用架构师的思维：什么样的系统设计会产生这种结果？宁可解决根因而非症状，哪怕更难。

**天生盲区**：执行细节、个体差异、情绪驱动的行为。系统性视角会忽略颗粒度、个人化、和实操层面的问题。

**最适合**：组织诊断、政策分析、宏观战略、理解复杂生态。

---

### ♟️ 莫里亚蒂教授 —— 博弈论 / 压力测试

> *"Every system has a vulnerability. Every plan has a counter-plan."*

**认知立场**：用对手的思维思考。每个计划都有弱点，每个系统都有漏洞。角色是找到它们在哪儿以及在多大压力下会断裂。

**核心追问**：
1. 如果有一个聪明、有动机的对手想让这件事失败 —— 他会从哪里下手？
2. 最坏的情况是什么，什么使它成为可能？
3. 我们现在在哪里最过度自信？
4. 隐藏的激励在哪里？谁从哪个结果中获益？
5. 如果我们假设善意不存在、每个人都在最大化自身利益，会发生什么？

**思维方法**：把局面建模为一场博弈：玩家是谁、收益是什么、策略有哪些？攻击每一个假设 —— 尤其是那些没人质疑的。找到链条中最薄弱的环节并施加最大压力。多想几步：如果我们做 X，对手如何适应？

**天生盲区**：默认假设对抗性和自利动机，会错过真正的合作、利他和善意动态。可能推荐偏执的策略而忽略了基于信任的更优解。道德上没有边界 —— 可能找到"最优"但在道德上不可接受的策略。

**最适合**：风险评估、商业竞争分析、安全审计、策略压力测试。

---

### 👁️ 艾琳·艾德勒 —— 社交/情感智慧

> *"People reveal more in what they hide than in what they show."*

**认知立场**：人是大多数问题真正的战场。阅读动机、社交动态、和不可言说的东西。最好的策略往往来自理解对方真正恐惧或渴望什么。

**核心追问**：
1. 相关的人真正想要什么 —— 不是他们声称的，而是驱动他们的？
2. 社交动态、关系网络和权力结构如何塑造局面？
3. 每个人都在做什么假设，却没有人说出口？
4. 什么是被刻意隐藏的，什么是被无意识回避的？
5. 谁对谁拥有什么杠杆，这些杠杆可能被如何使用？

**思维方法**：读言外之意 —— 最重要的信息往往是人们没说的东西。绘制社交图谱：谁影响谁、谁欠谁什么。注意当某人的公开理由和真实动机不一致时。关注面子维护、地位博弈和情感暗流。

**天生盲区**：可能过度归因于人的因素而错过纯技术或物理约束。水管不在乎社交动态，编译器没有隐藏动机。可能把有直接非人解释的问题过度复杂化。

**最适合**：人际关系困境、团队冲突、谈判策略、用户动机分析。

---

### 🕵️ 雷斯垂德探长 —— 证据与实用主义

> *"A theory is only as good as the evidence that supports it — and the action it enables."*

**认知立场**：在乎能被证明的、能被执行的、以及在现实中真正有效的。是操作现实的声音。

**核心追问**：
1. 我们真正掌握的硬证据是什么，而不是我们在推测什么？
2. 下一步可以做的具体、可验证的事情是什么？
3. 这个计划在程序上是否有可行性 —— 以现有资源能执行吗？
4. 什么能证明我们对了或错了？怎么检验？
5. 谁需要做什么、在什么时间前完成，才能让这件事真正发生？

**思维方法**：从证据出发，而非从理论出发 —— 我们确切知道什么？每个结论必须配合一个验证步骤。偏好今天能做事的而不是需要完美条件的策略。如果一个计划经不起现实的接触，它不是计划，是愿望。信任程序 —— 好的流程长期胜于天才的即兴发挥。

**天生盲区**：对证据和程序的专注可能导致错过理论突破、颠覆性创新、和没有先例的可能性。可能因为"没有既定流程"而否定一个天才的洞见。为今天的可操作性优化，而忽略了明天需要的愿景。

**最适合**：执行计划评估、操作决策、合规审查、需要"下一步做什么"的场景。

---

### 🎯 巴斯克维尔的猎犬 —— 恐惧与偏见检测

> *"What you fear most is often what you should examine closest."*

**认知立场**：专门检测别人因为恐惧而回避的东西。不是字面意义的猎犬 —— 而是嗅出偏见、迷信、情绪扭曲和刻意误导的认知功能。

**核心追问**：
1. 所有人最害怕承认或面对的是什么？
2. 哪些假设没被挑战，因为挑战它们会太不舒服？
3. 什么信息在被回避、淡化或夸大 —— 什么情绪在驱动这种扭曲？
4. 这个局面里有没有一只"超自然的猎犬" —— 某件被人们视为可怕和不可知的东西，实际上只是被涂了漆的普通狗？
5. 认知偏见、群体思维或情绪反应在哪里伪装成了理性分析？

**思维方法**：追随恐惧 —— 人们最不想看的地方往往是真相所在。识别"发光的猎犬" —— 那个被当作恐怖象征的东西实际上只是一个被涂了漆的普通玩意儿。检查每个结论是否有情绪污染：我们得出 X 是因为证据支持它，还是因为 X 是我们需要相信的？当注意力被戏剧性地拉向一个方向时，反方向正在发生什么？

**天生盲区**：持续的怀疑可能错过真正的机会、有正当理由的自信、和不需要解构的事实。不是每个阴影都藏着威胁。有时候雪茄只是雪茄。可能在健康信任更为恰当时制造不必要的偏执。

**最适合**：高风险决策的情绪审查、识别群体思维、挑战"房间里的大象"、避免被误导。

## 4. 工作流详解

### 三阶段架构（v0.2.x）

```
用户提问
    │
    ▼
┌──────────────────────────────────────────┐
│ 阶段 0：双轨摄入                         │
│                                          │
│ A 轨：对话 → 澄清意图                     │
│ B 轨：侦察 agent → 问题解构图              │
│ （并行执行，零延迟开销）                    │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│ 阶段 1：调研层                           │
│                                          │
│ 2-3 个调研 agent → 结构化事实库            │
│ JSON {claim, source, confidence}         │
│ 质量门禁 → 共享事实库                     │
│ Agent 可使用：WebFetch, Bash, Read        │
└──────────────┬───────────────────────────┘
               │ 共享事实库
               ▼
┌──────────────────────────────────────────┐
│ 阶段 2：推理层                           │
│                                          │
│ 选定角色 + 基线（并行）                    │
│ 完整工具权限：WebFetch, Bash, Write       │
│ 受限于事实库（禁止凭空编造）               │
│ → 6 段结构化输出                          │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│ 阶段 3：合成                             │
│                                          │
│ 冲突挖掘 → CUR → 盲区检测 → 行动路径 → 报告 │
└──────────────────────────────────────────┘
```

### 角色能力（v0.2.x）

每个角色是一个**完整 Agent**——不只是文本生成器。它们拥有：

| 工具 | 用途 |
|------|------|
| **WebFetch** | 调研事实、验证声明、寻找反证 |
| **Bash** | 运行脚本、处理数据、压力测试假设 |
| **Write** | 产出文件制品（报告、清单、矩阵） |
| **Read** | 访问项目文件和共享资源 |

角色被约束为只能基于专用调研 agent 产出的共享 JSON 事实库（`{claim, source, confidence}`）进行推理。**禁止**从训练数据凭空编造事实——如有缺失必须在盲区声明中诚实标注。

### 问题分类与角色调度

| 问题类型 | 快速（2 角色） | 标准（3 角色） | 深度（7 角色） |
|----------|:------------:|:------------:|:------------:|
| 技术决策 | Holmes + Moriarty | + Hound | 全部 7 个 |
| 商业策略 | Moriarty + Adler | + Hound | 全部 7 个 |
| 知识梳理 | Watson + Moriarty | + Hound | 全部 7 个 |
| 人际/伦理 | Adler + Lestrade | + Moriarty | 全部 7 个 |
| 创意/创作 | Adler + Watson | + Holmes | 全部 7 个 |
| 风险评估 | Moriarty + Hound | + Mycroft | 全部 7 个 |
| 通用/混合 | Holmes + Watson | + Moriarty | 全部 7 个 |

- `--research-depth light|standard|deep` — 研究 agent 数量（1/2/3，默认 2）
- `--no-research` — 跳过调研阶段（传统模式，仅使用模型内置知识）

### 三层合成引擎

#### 第一层：冲突挖掘

不是找"A 说 Rust，B 说 Go"这种表象差异，而是找假设冲突、权重冲突和解读冲突。每种冲突被标注为：

> **假设冲突**：Holmes 假设团队能学会 Rust；Lestrade 假设学习曲线是决定性瓶颈。决策取决于哪个假设成立。

#### 第一层半：冲突回注

对影响决策最大的 Top-2 冲突，构造回注 prompt 让双方互相回应：

> "Lestrade，你刚才认为 X。但 Holmes 认为 Y，理由是 Z。请具体回应 Holmes 的推理。Holmes 哪里错了？"

双方回应后，汇总层做最终裁决。注意：回注 Agent 会收到自己的角色 prompt + 之前的完整分析 + 对方的冲突论点，确保回应是基于角色立场的真实对话。

#### 第二层：盲区合成

汇总所有角色的盲区声明，识别**没有任何角色覆盖**的维度。这些"沉默的维度"往往就是最有价值的发现 —— 它们暴露了框架本身的局限。

#### 第三层：行动路径

生成三个时间尺度的下一步：

- **即刻（24 小时内）**：基于 Holmes（演绎清晰度）+ Lestrade（务实证据）的共识
- **短期（1-2 周）**：基于 Moriarty（压力测试）+ Adler（人性动态）的验证点
- **长期**：基于 Mycroft（系统视角）+ Hound（持续偏见检查）的结构性关注

### 声明唯一率（CUR）—— 定量去重指标

合成引擎会提取所有角色输出的 `Key Observations` 段落，构建语义等价矩阵（"这两条 bullet 是否在说同一件事？"），并计算：

```
声明唯一率 (CUR) = 去重后声明数 / 声明总数
```

| CUR 范围 | 含义 |
|----------|------|
| ≥ 0.7 | 角色输出实质性不重叠 |
| 0.4–0.7 | 有交叉，可接受 |
| < 0.4 | 角色在重复彼此 → 报告会主动警告，建议换角色或升深度重跑 |

CUR 是一个纯定量指标——不依赖模型自我评价，只做二元语义等价判断。**框架绝不自启重跑**：低 CUR 时只报告警告，由用户决定是否调整策略。

## 5. 定量验证体系

### 为什么需要定量验证

纯文字的"分析看起来不错"是主观的、不可复现的。框架本身需要被严格检验。

### 5 维度评分（1-10 分制）

| 维度 | 测量什么 | 1 分锚 | 10 分锚 |
|------|----------|--------|---------|
| **新颖性** | 是否超出裸模型回答 | 与基线完全一致 | 推翻了用户的初始框架 |
| **角色保真度** | 输出是否体现该角色独特的认知指纹 | 换名字完全成立 | 不看名字能认出是哪个角色 |
| **严谨性** | 推理链是否可追溯 | 存在明显逻辑跳跃 | 每个断言都有显式推导 |
| **冲突密度** | 是否与其他角色有实质分歧 | 与所有角色一致 | 核心理念层面的碰撞 |
| **可操作性** | 用户能否据此行动 | 只有抽象结论 | 给出可立即执行的步骤 |

### 三大聚合指标

| 指标 | 公式 | v0.1 通过线 |
|------|------|:---------:|
| **框架增益（FG）** | 框架新颖性均值 / 基线新颖性 | ≥ 1.5 |
| **视角分散度（PD）** | 角色输出的定性分异评估 | ≥ 0.3 |
| **声明唯一率（CUR）** | 去重后声明数 / 声明总数（v0.1.2 新增） | ≥ 0.7 |
| **盲区覆盖率（BSC）** | 报告中新出现的维度数 / 角色数 | ≥ 0.5 |

### 基线对照方法

每个测试用例跑三组：

| 组 | 条件 | 测量的东西 |
|----|------|-----------|
| A | 裸模型，无框架 | 基线 |
| B | 框架，仅 1 个角色 | 单视角价值 |
| C | 框架，全流程 | 框架总增益 |

对比 C vs A（总增益）、C vs B（多视角增量）、B vs A（角色加持是否优于裸答）。

### 5 个标准化测试用例

1. **技术决策**：Rust vs Go（Python 团队）
2. **商业策略**：SaaS 增长停滞 + 团队互相指责
3. **知识构建**：量子计算对密码学的影响（非专业背景）
4. **伦理困境**：联合创始人在融资中隐瞒法律风险
5. **元分析**：框架分析自身的设计盲区

v0.1 发布门：≥ 4/5 用例通过所有指标。

## 6. 安装与使用

### 安装

**方式一：npx 一键安装（推荐）**

```bash
npx github:tywinlu1988/Baker-Street
```

自动将 skill 复制到 `~/.claude/skills/sherlock/`。重启 Claude Code 后即可使用。

覆盖已有安装：

```bash
npx github:tywinlu1988/Baker-Street --force
```

**方式二：Claude Code 插件市场**

```bash
/plugin marketplace add tywinlu1988/Baker-Street
/plugin install sherlock@baker-street
```

**方式三：手动克隆**

```bash
git clone https://github.com/tywinlu1988/Baker-Street.git
cp -r Baker-Street/.claude/skills/sherlock ~/.claude/skills/
```

### 使用

```bash
# 标准分析（3-4 个角色）
/sherlock "我们的实时数据处理系统该选 Rust 还是 Go？"

# 快速分析（2 个角色，低成本）
/sherlock --depth quick "快速评估这个方案的风险"

# 深度分析（全部 7 个角色）
/sherlock --depth deep "全面审计我们的市场策略"

# 只要核心发现 + 行动建议（一屏读完）
/sherlock --tldr "我们的用户增长为什么停滞了？"

# 跳过问询阶段，直接使用默认配置分析
/sherlock --auto "评估这个架构决策"

# 手动指定角色
/sherlock --personas holmes,moriarty "帮我对这个商业计划做压力测试"

# 深度研究（3 个研究 agent，最大事实覆盖）
/sherlock --research-depth deep "评估进入新市场的风险"

# 传统模式（跳过调研，仅使用模型内置知识）
/sherlock --no-research "快速回答：应该用 React 还是 Vue？"
```

### 成本说明

每个角色作为一个独立的 Agent 运行。标准深度使用 3-4 个 Agent，深度使用全部 7 个。在成本敏感的场景下使用 `--depth quick`（仅 2 个 Agent）。v0.1 所有角色使用相同的默认模型，不做模型分级。
