## Well-Known Universal Facts (usable without citation per persona rules)
- WebAssembly (WASM) shipped in browsers 2017
- Designed as binary compilation target, complement to JS
- Allows C/C++/Rust/Go/Java to run in browsers
- JavaScript remains the native web scripting language
- WASM has garbage collection (GC) proposal recently standardized
- WASM has no direct DOM access; must go through JS bridge
- Component Model proposal aims to improve interoperability
- WASM used in production by: Figma (design tool), Cloudflare (Workers edge compute), Google Earth, AutoCAD (web version)
- WASM is NOT a 1:1 replacement for JS DOM manipulation
- WASM startup/instantiation overhead exists but shrinking
- WASM typically outperforms JS on computation-heavy tasks
